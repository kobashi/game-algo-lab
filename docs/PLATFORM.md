# 共通基盤仕様（教材プラットフォーム）

最終更新: 2026-07-17

全トピックで揃える **見た目・操作・ファイル配置・用語**。  
新しいデモを足す人は、ここを満たすことを目標にする（アルゴリズム固有の部分だけが差分になる）。

---

## 1. 技術制約

| 項目 | 規約 |
|------|------|
| 配布 | 静的サイト。GitHub Pages（`main` ルート） |
| 言語 | HTML / CSS / ES modules（JS）。ビルド不要 |
| ローカル確認 | `python3 -m http.server 8080`（`file://` 非推奨） |
| C# サンプル | `samples/`。デモと 1:1 厳密対応は不要だが、トピックの核は示す |
| 依存 | 外部 CDN や npm は原則使わない（追加するなら SPEC で理由を書く） |

---

## 2. ディレクトリ規約

```
algorithms/<id>.html     # デモページ
js/<id>.js               # デモロジック（ES module）
js/maps/<id>-map.js      # 初期データ（経路探索系）※トピック種別に応じて置き場を SPEC で決めてよい
samples/<PascalCase>Example.cs
css/style.css            # 共通スタイル（トピック専用 CSS は最小限）
js/ds-viz.js             # データ構造パネル共通
js/map-format.js         # グリッド地図パーサ（経路探索系）
js/main.js               # トップの TOPICS 定義
docs/topics/<id>/SPEC.md # トピック仕様（実装前に作成）
```

- **id**: 英小文字・ハイフン（例: `alpha-beta`, `best-first`）  
- トップへの掲載: `js/main.js` の `TOPICS` に1件。未完成は `ready: false`

---

## 3. 学習体験の統一

どのデモも、だいたい次の骨格にする。

1. **タイトル + 短いリード**（何を学ぶか1〜2文）  
2. **可視化キャンバス / 盤面**（メイン）  
3. **操作**（再生 / 1ステップ / リセット系）— 盤面と同じ「ゲーム群」に置く  
4. **データ構造パネル**（キュー・スタック・集合・木など）  
5. **折りたたみ解説**（`<details>`）  
6. **C# サンプル**（`<pre>` または fetch）  
7. **結果・比較**（任意。弱点の可視化など）

### レイアウト

- **縦長**: ゲーム群の下に DS  
- **横長**: 左ゲーム群 | 右 DS  
- クラス名の慣習: `.demo-game-group`, `.ds-section`, `.demo-controls`  
- 操作パネル自体はスクロール不要。長い結果比較だけスクロール可  

### 操作行（経路探索デモの標準）

| 行 | 内容 |
|----|------|
| 1 | 再生 / 1ステップ / 地図再読込、速度、式（あれば） |
| 2 | 塗りツール（コスト・壁・ゴールなど） |

トピックがグリッドでない場合（ゲーム木など）は、**同等の役割**（再生・ステップ・リセット・パラメータ）を同じ位置感で置く。

## 4. 経路探索系の共通データ

### 地図記号（`js/map-format.js`）

| 記号 | 意味 |
|------|------|
| `S` | スタート（1つのみ） |
| `G` | ゴール（**複数可**。いずれか到達で成功） |
| `#` | 壁 |
| `.` | コスト 1 |
| `0` `1` `2` | 通行コスト |
| `n` / `-` | コスト -1 |

- 初期地図の正本は **エディタ上の `js/maps/*-map.js`**  
- ブラウザの塗りは「試す用」。再読込でファイル内容に戻る  

### ペイント

- モード: コスト / 壁 / G（複数ゴールの追加・削除）  
- スタート上には書けない  
- ゴールは最低1つ  

### ヒューリスティック（複数ゴール時）

- マンハッタン等は **最も近い G まで** の距離を使う  

---

## 5. データ構造可視化（`ds-viz.js`）

使える部品の例:

- キュー / 集合 / 優先度付きオープン  
- 親ポインタ: リスト + ツリー（HTML にタブ固定、中身だけ更新）  
- コールスタック（DFS など）  

新規 DS 表現が必要なら `ds-viz.js` に汎用 API を足し、トピック固有の描画は `js/<id>.js` に閉じる。

---

## 6. 用語（教材内で揃える）

| 用語 | 意味 |
|------|------|
| 歩数 | マスを何回進んだか（BFS が最小化する対象） |
| 経路コスト g | 通過マスのコスト累積 |
| ヒューリスティック h | ゴールまでの見積り |
| f | g + h（A*） |
| オープン / クローズド | 未展開候補 / 展開済み |
| バックトラック | 行き止まりから戻る |

新しい用語は SPEC の「用語」節に定義する。

---

## 7. UI コピー（文言）のトーン

- です・ます調、短文  
- 警告は「⚠」＋原因と次に試す操作のヒント  
- 状態ログは1行で状況がわかる（座標、g、キュー長など）  

---

## 8. アクセシビリティ（最低ライン）

- ボタンに意味のわかるラベル  
- 塗りツールは `aria-pressed`  
- ステータスは `role="status"`  

---

## 9. プラットフォーム変更の扱い

`css/style.css` / `ds-viz.js` / `map-format.js` / レイアウト骨格を変える PR は:

1. 既存の `ready: true` デモを壊さない  
2. 本ドキュメントを同じ PR で更新する  
3. WORKFLOW の「基盤変更」チェックを通す
