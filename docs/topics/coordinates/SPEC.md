## 1. 学習目標

1. **ローカル / ワールド / スクリーン**の3座標系を区別して説明できる  
2. 親子の位置・回転から **ローカル→ワールド**（合成）と **ワールド→ローカル**（逆変換）ができることを観察できる  
3. ワールド→スクリーンは **平行移動 + スケール**（簡易カメラ）で表せると分かる  

---

## 2. なぜゲームで使うか

キャラの手元の武器・エフェクトはローカル座標で置き、ワールドに載せて動かし、最後にカメラ経由で画面に描く。ヒット判定や UI ピックは座標系の取り違えがバグになりやすい。

---

## 3. アルゴリズム概要（2D）

点 `p`、親の位置 `T`、回転 `θ`、スケール `s`（本デモ親スケール=1）:

```
// ローカル → ワールド
world = Rotate(θ) * local + parentWorldPos

// ワールド → ローカル
local = Rotate(-θ) * (world - parentWorldPos)

// ワールド → スクリーン（カメラ）
screen = (world - cameraPos) * zoom + screenCenter
```

デモ構成:
- **親**（プレイヤー）: ワールド上をドラッグ or スライダーで移動・回転  
- **子**（手元マーカー）: 親ローカル座標 `(lx, ly)`  
- クリックしたスクリーン点をワールド→ローカルに落として表示  

---

## 4. 操作

| 操作 | 挙動 |
|------|------|
| 親の移動・回転 | スライダー or キャンバスクリックで親位置 |
| 子ローカル XY | スライダー |
| カメラ XY / zoom | スライダー |
| キャンバスクリック | その点の screen→world→local を表示 |
