## 1. 学習目標

1. **OR 節点**は「どれか1つ成功すればよい」（自分の選択・存在量化）と説明できる  
2. **AND 節点**は「すべて成功が必要」（相手の手・偶然の全分岐・全称量化）と説明できる  
3. 再帰的に木を解く過程を、コールスタックと節点の真偽ラベルで追える  

---

## 2. なぜゲームで使うか

完全情報の読み切り・クエスト分岐・「全パターン耐えられるか」といったモデルの入口。  
Min-Max は「数値評価付きの AND/OR（max/min）」と位置づけ、本トピックがその前段。

---

## 3. アルゴリズム概要

- 入力: 根付き木。各節点は `OR` / `AND` / 終端（真 or 偽）  
- 出力: 根が解けるか（真偽）と、証明に使った部分木のハイライト  
- 停止: 終端に到達、または子の結果から節点の真偽が確定  

```
Solve(n):
  if n が終端: return n.value
  if n が OR:
    for c in children(n):
      if Solve(c): return true
    return false
  if n が AND:
    for c in children(n):
      if not Solve(c): return false
    return true
```

探索順: 深さ優先・子は左から右（教材固定）。

## 4. 画面と操作

### 可視化

- ゲーム木（SVG）: 節点ラベル、種別（OR/AND/終端）、状態色  
  - 未評価 / 評価中 / 真 / 偽  
- 証明に寄与した枝を強調  

### 必須操作

| 操作 | 挙動 |
|------|------|
| 再生 | 自動ステップ |
| 1ステップ | 再帰1動作（入場・子へ・結果確定・return） |
| リセット | 初期木に戻し未評価状態 |

### データ構造パネル

- コールスタック（`Solve(節点)` のフレーム）  
- 確定済み節点の一覧（真/偽）

---

## 6. 成功 / 失敗

- 根が真 → 「解あり（クリア戦略が存在する）」  
- 根が偽 → 「解なし（どの選択でも条件を満たせない）」
