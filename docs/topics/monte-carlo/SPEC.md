## 1. 学習目標

1. **プレイアウト**（根から葉まで乱択で進む）の平均で局面を評価する流れを追える  
2. Min-Max の「最善前提の値」と、乱択の**期待値**は一般に違うと説明できる  
3. 試行を増やすと推定が安定する（ばらつきが減る）様子を観察できる  
4. 次の多腕バンディット／MCTS へつながる「サンプルで決める」発想を掴む  

---

## 2. アルゴリズム（本デモ）

単純モンテカルロ評価（ランダムプレイアウト平均）:

```
Estimate(root, N):
  sum = 0
  for i in 1..N:
    n = root
    while n が葉でない:
      n = children(n) から一様ランダム
    sum += n.score
  return sum / N
```

- 乱数シード固定オプションで再現可能にする  
- 真の乱択期待値 EV と Min-Max 値を比較表示

## 3. 初期木

Min-Max・α-β と同型・同値の深さ3・葉12標準木（`js/maps/monte-carlo-tree.js`、2026-07-19 改訂）。
深さ3化によりプレイアウトが3手の系列として見え、1ステップ=経路1マスの設計が意味を持つ。

- Min-Max 最善: 手 R、v=7  
- 乱択 EV: 各枝を等確率で進んだときの葉の平均 ≈ 4.92（手 L≈6.0 と手 R≈6.25 が僅差になり、
  Min-Max の 5 vs 7 とは食い違う点が教材のポイント）
