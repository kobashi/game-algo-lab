## 1. 学習目標

1. **シード付き PRNG** は同じシードなら同じ乱数列を返すことを確認できる  
2. **Mulberry32 / XorShift32 / LCG** を切替えて系列・ヒストグラムの違いを観察できる  
3. **LCG** の `a,c,m` が周期の上限（≤ m）と質に効くこと、質の悪い例と許容例を体感できる  

---

## 2. なぜゲームで使うか

プロシージャル生成・ドロップ・AI の乱択は、バグ再現や「同じマップを友達と共有」のためにシードが要る。  
実装を「ただの LCG」にすると周期が短く、マップがすぐループする — パラメータ選定の重要性を示す。

---

## 3. アルゴリズム概要

| id | 式・要点 |
|----|----------|
| `mulberry32` | サイト標準（`js/platform/rng.js`） |
| `xorshift32` | `x^=x<<a; x^=x>>>b; x^=x<<c`。プリセット: 悪い (1,1,1)/(5,0,5)、標準 (13,17,5)、代替 (7,9,13) |
| `lcg` | `X'=(a·X+c) mod m`、`u=X/m` |

LCG 周期計測: m ≤ 65536 のとき状態を辿り初期状態に戻る長さを数える。

### LCG プリセット（`LCG_PRESETS`）

| 質 | 例 |
|----|-----|
| 悪い | m=16 極小、a=2,c=0,m=256 |
| 許容（教材） | m=31 / m=64 で周期を数え切れる |
| 実用寄り | glibc 風 m=2^31、NR 風 m=2^32（周期は未全列挙） |
