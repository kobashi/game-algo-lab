## 1. 学習目標

1. **シード付き PRNG** は同じシードなら同じ乱数列を返すことを確認できる  
2. シードを変えると系列が変わり、**再現性**（デバッグ・リプレイ）に使えると説明できる  
3. `Math.random()` と異なり、教材デモでは **明示シード** が比較実験の前提だと分かる  

---

## 2. なぜゲームで使うか

プロシージャル生成・ドロップ・AI の乱択は、バグ再現や「同じマップを友達と共有」のためにシードが要る。テストでは乱数を固定して決定的にする。

---

## 3. アルゴリズム概要

本サイト共通: **Mulberry32**（`js/platform/rng.js`）

```
seed = 32bit 整数
next() → [0, 1) の一様乱数
```

デモ:
- シード入力 → N 個の乱数を生成しリスト / 棒グラフ
- 「同じシードで再生成」→ 完全一致
- 「シード+1」→ 別系列
- 簡易: 0..1 をコイントス / 整数ダイスに変換

---

## 4. 操作

| 操作 | 挙動 |
|------|------|
| 生成 | 現シードから N 個サンプル |
| 同じシードで再生成 | 系列が一致することを示す |
| シードをずらす | +1 して再生成 |
| N / シード | 数値入力 |
