# トピック成熟度（修正状況）

最終更新: 2026-07-17  

各トピックが **一発で出したままか / 改訂を経たか / 授業用に据え置けるか** を揃えて記録する。  
**正本の一覧は [CATALOG.md](./CATALOG.md)**。本ファイルは **ラベルの定義と昇格条件**。

---

## 成熟度ラベル（3段階）

| コード | 表示名 | 意味 |
|--------|--------|------|
| `oneshot` | **一発未調整版** | 要件どおり一通り実装した段階。大きな教材改訂・UI 再設計はまだない（one-shot） |
| `revised` | **改訂・調整版** | 公開後または実装後に、学習目標・操作・表示・バグ・基盤寄せなどで **意図した改訂** が入っている |
| `stable` | **安定版** | 授業・試作運用で据え置き可能と判断。**破壊的な仕様変更は避ける**（直すならパッチ級か次メジャーで明示） |

`ready: true`（メニューでデモを開ける）とは **別軸**。

| | oneshot | revised | stable |
|--|---------|---------|--------|
| デモが動く (`ready`) | 多い | 多い | 必須 |
| 改訂履歴がある | ほぼなし | あり | あり（据え置き決定含む） |
| 破壊的変更 | 自由 | 可（理由を残す） | 原則しない |

---

## 何を「改訂」と数えるか

**改訂（oneshot → revised）の例**

- 学習上の弱点・比較 UI の追加（例: BFS のコスト比較、AABB の二重判定）
- 操作・ペイント・複数ゴールなど機能の追加・直し
- 説明レイアウトの作り直し（マップ流用をやめる等）
- バグ修正が教材の正しさに効くもの（探索結果・式の誤り）

**改訂に含めない（基盤共通化だけ）**

- ヘッダー/フッターの `topic-shell` 化だけ
- `createPlayback` への寄せだけ  
  → 挙動が同じなら **oneshot のまま**でよい（注記「基盤のみ更新」可）

**安定版（revised → stable）にするとき**

- [ ] 学習目標が SPEC / 画面で一通り辿れる  
- [ ] 主要な操作（再生・1ステップ・リセット）が破綻しない  
- [ ] 既知の致命バグがない（または注記済み）  
- [ ] 授業担当が「この版で進める」と決めた  
- [ ] CATALOG の成熟度を `stable` に更新し、日付を残す  

---

## 更新ルール

1. 成熟度を変えたら **CATALOG.md を同じ PR/コミットで更新**する  
2. 可能なら `docs/topics/<id>/SPEC.md` の表に `成熟度` を書く  
3. 大きな改訂のたびに CATALOG の「改訂メモ」列を1行足す（短く）  
4. `js/main.js` の `maturity` は CATALOG と揃える（トップに表示する場合）

## 初期分類の考え方（v0.9.0 試作時点）

試作 Release 時点のスナップショット。以後の改訂で随時更新する。

| 成熟度 | 方針 |
|--------|------|
| 経路探索 5 本 | 複数ゴール・ペイント・表示・platform 寄せなど改訂あり → **revised** |
| ゲーム木 5 本 | シリーズ実装が中心。内容の大きな作り直しは少ない → **oneshot**（基盤寄せのみのものは oneshot 維持） |
| AABB | 二重判定・複雑度比較の改訂あり → **revised** |
| FSM | 一通り実装 → **oneshot** |
| stable | 試作段階では **なし**（授業確定後に昇格） |
