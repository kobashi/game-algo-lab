AABB 衝突判定

軸に平行な矩形（AABB）同士の当たり判定。 重なり判定（ポジティブ）と分離判定（ネガティブ）の2通りを並べ、 結果の一致とコードの複雑度を比較する教材です（経路マップは使いません）。

このデモで学ぶこと
  • AABB = 軸平行バウンディングボックス
  • A. ポジティブ: overlapX ∧ overlapY（重なりを積み上げる）
  • B. ネガティブ: ¬(separatedX ∨ separatedY)（分離していなければ衝突）
  • ド・モルガンで同値。早期 return 版はゲーム実装でよく使われる
  • 下の表で比較回数・論理演算・分岐の目安を比較できる

青 = プレイヤーの当たり判定、橙 = 障害物。ドラッグで動かして交差を観察。

2 通りの判定ロジック

同じ入力に対し、A と B は常に同じ真偽を返します（境界 inclusive の定義を揃えた場合）。 違いは「何を肯定するか」と、実装したときの読みやすさ・早期打ち切りのしやすさです。

A. ポジティブ（重なり判定）

—

「両軸で重なっている」ことを積み上げる。

  1. ?
    ① overlapX A.maxX ≥ B.minX ∧ A.minX ≤ B.maxX
  2. ?
    ② overlapY A.maxY ≥ B.minY ∧ A.minY ≤ B.maxY
  3. ?
    ③ 衝突 return overlapX && overlapY
